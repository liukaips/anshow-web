# AnShow 管理后台运营化重构设计

## 1. 目标

将现有以技术字段和零散表单为中心的 Admin，重构为面向非技术运营人员的中文 B 端管理系统。用户不需要理解 `code`、`slug`、`draft`、权限标识、图片焦点坐标等内部概念，也能完成内容创建、多语言确认、审核发布、整站预览、询盘跟进、媒体管理、员工权限、审计和系统配置。

本次不是单纯换色或调整间距，而是同时重构：

- 后台信息架构和页面骨架；
- 内容创建、翻译、审核、预览和发布工作流；
- 表格、筛选、表单、状态和反馈组件；
- 询盘、审核、审计等尚未完整交付的运营模块；
- 后端支持的审批状态、预览快照、限时访问令牌和审计数据。

## 2. 已确认的产品决策

- 采用 A 型“传统企业管理台”：固定侧栏、统一顶栏、页面标题、筛选工具栏和标准数据表格。
- 主要用户是非技术运营人员，默认界面不得要求其填写内部编码或技术参数。
- 多语言采用中文主录入，系统自动生成英文和俄文，运营人员可逐语言修订和确认。
- 发布采用角色化双模式：有权限的管理员可以直接发布，普通编辑提交审核后由审核人员发布。
- 补齐询盘管理、审核中心、审计日志和工作台统计，而非只重构已有页面样式。
- 预览是完整前端网站快照，不是 Admin 内嵌的单条内容预览。
- 预览使用官网同域名路径：`/preview/{token}/{locale}/...`，不新增 DNS 或预览子域名。
- 预览支持后台登录访问和限时外部分享链接；链接可撤销、可过期、不可编辑。

## 3. 设计原则

### 3.1 业务语言优先

- 页面只展示中文业务术语，例如“草稿”“待审核”“已发布”“翻译待确认”。
- 内部编码、网址片段和 SEO 字段由系统自动生成，放入折叠的“高级设置”。
- 权限以“内容编辑”“内容审核”“系统管理员”等角色模板呈现，不展示权限字符串。
- 媒体焦点通过点击图片选择，不展示 `focalX`、`focalY` 数值输入。

### 3.2 操作结果明确

- 保存、自动翻译、提交审核、发布、退回、归档、删除均提供明确的处理中、成功和失败反馈。
- 错误显示在对应字段或操作区域，说明原因和恢复办法。
- 危险操作优先使用归档；永久删除必须展示影响范围并二次确认。
- 离开未保存页面时继续拦截；自动保存成功后清除未保存提示。

### 3.3 稳定高密度布局

- 桌面以 1280 至 1920 像素为主要操作范围，保持清晰但不过度卡片化。
- 固定侧栏宽度 232px，内容列使用 `minmax(0, 1fr)`，避免内容被挤出视口。
- 顶栏高度 64px；页面横向内边距 24px 至 32px；表格行高不低于 52px。
- 控件最小高度 40px，核心操作和移动端触控目标不低于 44px。
- 移动端侧栏变抽屉，表格转换为分组列表，不强制横向滚动。

### 3.4 正式 B 端视觉

- 深色侧栏、白色工作区、浅灰页面背景。
- 青绿色作为主操作和成功状态；橙色只用于待处理提醒；红色只用于错误和危险操作。
- 圆角控制在 4px 至 6px；避免大面积渐变、装饰性阴影和营销式大标题。
- 页面标题采用 24px 至 30px，表格与表单使用适合长时间操作的紧凑字号和行高。

## 4. 信息架构

### 4.1 工作

- 工作台：我的待办、优先级、统计、最近动态和系统状态。
- 审核中心：待审核内容、三语言对比、退回原因、定时发布和批准发布。

### 4.2 内容

- 页面
- 首屏轮播
- 服务
- 贸易航线
- 特种货物
- 案例
- 文章
- 合作伙伴
- 资质证书
- 证明指标
- 导航项目
- 发布中心：汇总本次待发布改动，生成整站预览快照。

### 4.3 业务

- 询盘管理
- 媒体资源

### 4.4 系统

- 员工与权限
- 审计日志
- 系统设置

侧栏分组和所有菜单使用中文。无权限的模块不显示，当前路由使用左侧色条和背景同时标识。

## 5. 后台壳层

`AdminShell` 负责唯一的后台页面骨架：

- 桌面固定侧栏；
- 移动端抽屉导航；
- 顶栏面包屑、帮助入口、账号菜单和退出操作；
- 主内容区稳定宽度和滚动；
- 全局消息、确认对话框和页面离开保护。

侧栏自身允许纵向滚动，但不得与主内容产生横向滚动条。Logo、分组标题、菜单图标和文字必须完整显示，不允许截图中出现的裁切和错位。

## 6. 通用组件规范

重构时建立后台专用组件，不与公共官网的营销组件混用：

- `AdminPageHeader`：面包屑、标题、说明和主操作。
- `AdminToolbar`：搜索、状态筛选、负责人筛选、重置、批量操作。
- `AdminDataTable`：表头、固定列、状态标签、分页、空状态和加载状态。
- `AdminResponsiveList`：移动端表格替代布局。
- `AdminStatusBadge`：中文状态与语义颜色。
- `AdminFormField`：标签、必填、提示、字数、错误和恢复建议。
- `AdminSelect`、`AdminToggle`、`AdminStepper`：业务化输入控件。
- `AdminDrawer`：列表详情、引用关系和快速处理。
- `AdminDialog`：危险操作和提交审核确认。
- `AdminToast`：非阻塞成功反馈。
- `AdminEmptyState`：解释为什么为空以及下一步操作。
- `TranslationProgress`：中文、英文、俄文的生成和确认状态。
- `PublishTimeline`：草稿、翻译、审核、预览、发布进度。

## 7. 内容列表

列表不再以内部 code 为主标题。默认列为：

- 内容名称；
- 负责人；
- 多语言进度；
- 发布状态；
- 最近更新时间；
- 下一步操作。

支持按名称、负责人、发布状态、翻译状态搜索和筛选。内部编码只在详情抽屉或高级设置中可复制查看。

创建内容时，用户点击“新建服务”“新建文章”等业务按钮，直接进入四步编辑流程。系统根据中文标题生成唯一 code 和 slug；冲突时自动追加短后缀，不要求用户处理。

## 8. 内容编辑流程

### 8.1 第一步：填写中文内容

- 服务名称或页面标题；
- 一句话介绍；
- 详细说明；
- 按不同内容类型显示必要的业务字段。

系统实时生成网址和 SEO 默认值，并展示中文提示。高级设置允许有权限人员调整网址、SEO、排序和关联流程，但默认折叠。

### 8.2 第二步：选择展示图片

- 从媒体库选择或直接上传；
- 自动生成 AVIF/WebP 响应式衍生图；
- 点击图片选择视觉焦点；
- 中文图片说明必填，英文和俄文说明可随翻译一起生成。

### 8.3 第三步：确认自动翻译

- 后端通过 `TranslationProvider` 调用 OpenAI 兼容的翻译接口；使用原生 `fetch`，不新增前端依赖。
- 环境变量配置接口地址、模型和密钥，密钥永不返回浏览器。
- 英文和俄文分别显示“等待生成”“生成中”“待确认”“已确认”“生成失败”。
- 用户可以重新生成单个语言，也可以直接人工修改。
- 翻译服务不可用时保留中文草稿并给出重试入口，不阻止保存。

### 8.4 第四步：预览并提交

- 展示字段完整性、图片状态、翻译状态和风险提示。
- 编辑人员执行“提交审核”。
- 具备直接发布权限的管理员可选择“提交审核”或“进入发布中心”。
- 所有提交操作基于内容版本号，检测并发修改；冲突时要求刷新比较，不静默覆盖。

编辑器每隔固定时间和关键字段失焦时自动保存草稿，同时保留明确的“保存草稿”按钮。

## 9. 审核中心

审核队列支持按提交人、内容类型、等待时长和优先级筛选。详情区并排显示中文、英文和俄文，突出与上次发布版本的差异。

审核操作：

- 退回修改：必须填写中文原因，通知提交人并进入“需修改”。
- 批准：将内容标记为已批准，等待进入发布中心。
- 定时发布：选择未来时间并显示时区。
- 批准并发布：仅直接发布角色可用。

所有审核和发布操作写入审计日志。

## 10. 完整前端网站预览

### 10.1 预览目标

预览不是单条内容组件，也不是 Admin 内嵌 iframe。系统将当前待发布改动生成不可变的网站快照，并在新标签页打开完整公共前端。

预览路径：

```text
/preview/{rawToken}/{locale}/...
```

例如：

```text
/preview/pv_xxx/zh
/preview/pv_xxx/zh/services
/preview/pv_xxx/zh/services/cold-chain-logistics
```

预览页面可以正常使用导航访问首页、列表页、详情页、联系页和法律页面。顶部仅保留一条窄的“网站预览环境”标识，可返回 Admin、切换语言和查看改动清单。

### 10.2 快照数据

生成快照时冻结：

- 所有本次待发布内容及三语言翻译；
- 站点设置、导航和联系方式；
- 所引用媒体及焦点信息；
- 快照创建人、创建时间和内容版本。

草稿内容覆盖对应线上内容；未修改内容读取当时的已发布版本并写入快照，保证预览期间页面不会因其他人继续编辑而变化。

审核通过后发布的是快照记录的内容版本。如果快照之后源内容发生变化，系统要求重新生成预览和重新审核。

### 10.3 安全

- 数据库只存令牌哈希，不存原始分享令牌。
- 内部预览要求后台会话；外部预览使用随机高熵令牌。
- 外部链接可设置 1 小时、24 小时或 7 天有效期，并可随时撤销。
- 响应设置 `noindex`、`noarchive` 和 `Cache-Control: private, no-store`。
- 预览接口只读，不提供任何编辑或发布能力。
- 创建、访问、撤销和过期事件写入审计日志。

## 11. 工作台

工作台替换当前空白欢迎页，显示：

- 新询盘数量和高优先级数量；
- 待审核内容；
- 翻译待确认内容；
- 本周已发布数量；
- 我的待办；
- 最近操作动态；
- API、数据库、邮件、COS 和备份的简化健康状态。

统计只展示系统实际数据，不使用虚假业务指标。

## 12. 询盘管理

询盘列表字段：客户、公司、联系方式、起运地、目的地、运输方式、优先级、负责人、状态和更新时间。

业务状态统一为：

- 新询盘；
- 待跟进；
- 跟进中；
- 等待客户；
- 已完成；
- 已关闭。

详情抽屉支持分配负责人、记录跟进备注、修改优先级和状态、查看原始提交内容和邮件通知结果。列表支持 CSV 导出，导出操作写入审计日志。

## 13. 媒体资源

- 上传区与媒体列表分离，避免每张卡片内堆叠完整编辑表单。
- 默认网格显示预览、业务名称、使用位置和多语言说明状态。
- 点击资源打开详情抽屉，编辑中文/英文/俄文替代文本、视觉焦点和替换文件。
- 焦点使用图片上的可拖动标记，不暴露数值坐标。
- 删除前展示引用它的中文页面名称；存在引用时只允许替换或先解除引用。
- 图片尺寸、格式、字节数和内部 ID 放在“技术信息”折叠区。

## 14. 员工与权限

默认角色模板：

- 内容编辑：创建、编辑、翻译和提交审核；
- 内容审核：审核、退回和批准；
- 业务运营：处理询盘和媒体；
- 系统管理员：配置系统和员工；
- 超级管理员：全部权限。

界面使用中文权限说明，内部权限标识仅在技术信息中显示。禁用账号、重置密码、变更角色会撤销现有会话。系统继续保护最后一位启用的超级管理员，禁止禁用或降权。

## 15. 系统设置与审计

系统设置分为：

- 公司信息与公开联系方式；
- 询盘通知邮件；
- 媒体存储与腾讯云 COS 连接状态；
- 加密备份周期、保留期限和恢复验证；
- 翻译服务状态；
- 站点默认语言和功能开关。

密钥类字段只显示“已配置/未配置”，通过部署环境维护，Admin 不返回现有密钥值。

审计日志将技术操作转换为中文描述，支持按人员、模块、操作、结果和时间筛选。日志详情保留请求 ID、对象 ID 和变更摘要，敏感值必须脱敏。

## 16. 数据和 API 变化

新增或扩展的数据能力：

- 内容工作流状态、负责人、版本号和提交审核时间；
- 审核记录、退回原因、批准人和批准版本；
- 不可变预览快照、快照内容清单和令牌哈希；
- 询盘负责人、优先级、状态和跟进记录；
- 人类可读的审计事件元数据；
- 翻译任务状态和失败原因。

API 按领域分为内容、审核、预览、询盘、媒体、员工、设置和审计。所有写操作执行权限检查、输入校验、版本冲突检查和审计记录。

## 17. 错误处理

- API 不可用：页面保留骨架，显示中文错误、请求 ID 和重试按钮。
- 自动保存失败：保留本地表单值，顶部持续提示，恢复后允许重试。
- 并发冲突：展示服务器版本更新时间和当前编辑差异，禁止直接覆盖。
- 翻译失败：不影响中文草稿，允许按语言重试。
- 预览生成失败：不产生可访问链接，说明缺失内容或服务异常。
- 发布部分失败：发布事务回滚，不暴露半发布状态。
- 外部预览过期或撤销：显示中文失效页面，不回退到线上页面。

## 18. 可访问性与响应式

- 所有交互支持键盘操作和清晰焦点。
- 图标按钮有中文名称和 tooltip。
- 状态不能只依赖颜色，同时显示中文文字或图标。
- 表单错误使用 `aria-describedby` 和 `aria-live`。
- 桌面、平板和手机均不得出现无意义横向滚动。
- 用户启用减少动态效果时关闭非必要动画。

## 19. 测试策略

### 19.1 单元和组件测试

- 中文标签、状态映射和权限可见性；
- 自动 code/slug 生成和冲突处理；
- 多语言生成、确认和失败重试；
- 表格筛选、分页、移动列表和空状态；
- 表单校验、自动保存和离开保护；
- 媒体焦点选择和引用保护；
- 最后一位超级管理员保护。

### 19.2 后端集成测试

- 审核状态转换和角色权限；
- 内容版本冲突；
- 预览快照不可变性；
- 原始令牌不落库、过期和撤销；
- 草稿覆盖与已发布内容合并；
- 快照批准后原子发布；
- 询盘分配、跟进和审计；
- 密钥不出现在设置响应和日志。

### 19.3 E2E

- 编辑中文、生成翻译、确认、提交审核、退回、重新提交和发布；
- 从发布中心生成整站预览并导航所有主要公共页面；
- 外部预览链接访问、过期和撤销；
- 询盘分配和状态更新；
- 桌面 1280/1440/1920、平板 768/1024、手机 390 宽度无重叠和横向溢出；
- Admin 全页面中文扫描；
- 键盘和自动可访问性检查。

## 20. 验收标准

- Admin 所有用户可见操作、状态、帮助和错误均为中文。
- 非技术运营人员可以不填写任何内部编码完成内容创建和发布。
- 侧栏、顶栏、页面、表格和表单在目标宽度下对齐，无裁切和异常滚动条。
- 所有内容类型使用统一列表和编辑流程，不再出现页面各自为政的样式。
- 内容修改后可以生成完整前端网站快照，并在真实路由中检查最终样式。
- 普通编辑不能绕过审核；直接发布权限受角色控制。
- 询盘、媒体、员工、审计和设置达到可操作状态，不保留英文占位和内部字段直出。
- 所有新增写操作都有权限校验、错误恢复和审计记录。
